📖 Integration Guide

1. Device Pairing (Required)

AIWearable authenticates with OpenClaw via ED25519 device keys. On first boot, the device generates a keypair and stores it in NVS.

To pair:

  1. Note the Device Key shown above (or from serial log at boot: Device identity: <hex>)
  2. On the OpenClaw machine, run:
    openclaw device approve <device-key-hex>
  3. The device will auto-connect on next WebSocket attempt (every 5s)

⚠️ If using token auth instead of device keys, paste the token in the Token field above.

2. STT — DashScope (Required for voice)

AIWearable uses DashScope (百炼) real-time streaming ASR via WebSocket for speech-to-text.

Setup:

  1. Get an API key from DashScope Console
  2. Enter the API key in the Services tab
  3. Default endpoint: wss://dashscope.aliyuncs.com/api-ws/v1/inference/
  4. Default model: fun-asr-realtime-2026-02-28
3. TTS — MiMo (Optional — for voice responses)

For audio playback of responses, MiMo (Xiaomi) TTS is used via streaming SSE.

Setup:

  1. Get an API key from MiMo Platform
  2. Enter the API key and URL in the Services tab
  3. Default URL: https://api.xiaomimimo.com/v1/chat/completions
  4. Default model: mimo-v2-tts, voice: mimo_default
